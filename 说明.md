# 任务附件下载功能说明

本文只说明 `feature/task-attachment-download` 分支新增的能力，不替代项目原有 `README.md`。项目安装、认证、完整命令列表仍以原 README 和 `docs/commands.md` 为准。

## 功能目标

新增命令用于下载滴答清单任务中已经存在的附件。典型场景是：手机端把微信里的 Word、图片或其他文件转入滴答清单 inbox 后，电脑端通过 CLI 把附件拉回本机，继续做归档、OCR、转 Markdown 或行政材料处理。

该功能只读取滴答清单中的任务附件，不读取微信数据库，不读取 PC 微信缓存，也不涉及微信解密。

## 命令

```bash
dida attachment download \
  --project <project-id> \
  --task <task-id> \
  --attachment <attachment-id> \
  --output <file> \
  --json
```

参数含义：

| 参数 | 说明 |
| --- | --- |
| `--project` | 任务所在项目 ID，例如真实 inbox ID |
| `--task` | 任务 ID |
| `--attachment` | 附件 ID |
| `--output` | 保存到本机的文件路径 |
| `--force` | 可选。允许覆盖已经存在的本地文件 |
| `--json` | 可选。输出稳定 JSON envelope |

默认不会覆盖本地已有文件。如果输出路径已经存在，命令会失败并提示使用 `--force`。

## 示例

```bash
dida attachment download \
  --project inbox1011688506 \
  --task 6a24a837e4b0b53e82a80a8d \
  --attachment 6a24a837e4b0b53e82a80a8e \
  --output ./课题申请表.doc \
  --json
```

成功时返回示例：

```json
{
  "ok": true,
  "command": "attachment download",
  "data": {
    "attachmentId": "6a24a837e4b0b53e82a80a8e",
    "bytes": 89088,
    "contentType": "application/octet-stream;charset=UTF-8",
    "output": "C:\\path\\to\\课题申请表.doc",
    "projectId": "inbox1011688506",
    "taskId": "6a24a837e4b0b53e82a80a8d"
  }
}
```

## 如何找到 ID

先登录并确认认证可用：

```bash
dida auth login --browser --json
dida doctor --verify --json
```

读取上下文，找到真实 inbox ID：

```bash
dida agent context --outline --json
```

如果需要查看任务原始附件字段，可以用只读 raw 命令：

```bash
dida raw get /batch/check/0 --json
```

在对应任务中查找：

```json
{
  "id": "<task-id>",
  "projectId": "<project-id>",
  "attachments": [
    {
      "id": "<attachment-id>",
      "fileName": "课题申请表.doc",
      "size": 89088
    }
  ]
}
```

## 实现边界

本功能只覆盖“下载已存在的任务附件”。

暂不支持：

| 能力 | 状态 |
| --- | --- |
| 上传任务附件 | 不支持 |
| 把本地文件关联到任务附件 | 不支持 |
| 预览附件 | 不支持 |
| 删除附件 | 不支持 |
| 下载评论附件 | 不支持 |
| 自动解析微信内容 | 不支持 |

评论附件上传仍走已有 `dida comment create --file` 能力。任务附件的上传、关联、预览和删除需要单独验证接口、回滚路径和失败行为后再考虑实现。

## 安全说明

命令复用 dida-cli 现有本地 cookie 登录机制，不打印 token。请不要把 `~/.dida-cli/`、cookie、OAuth secret 或 MCP token 写入 Git、issue、PR 或聊天记录。

远端操作是只读下载。本地操作会写文件，因此默认拒绝覆盖已有文件。只有明确传入 `--force` 时才覆盖。
