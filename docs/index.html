Small surface, serious automation.

DidaCLI keeps the commands boring and the behavior inspectable, so a human can run it comfortably and an agent can recover from errors.

Agent-ready JSON

Every machine path returns a predictable envelope with `ok`, `command`, `meta`, `data`, or `error`.

Web API coverage

Reads sync, settings, projects, folders, tags, completed history, and writes through explicit resource endpoints.

Practical safety

Create, update, and complete run directly. Delete requires `--yes`; all writes support `--dry-run`.

Designed for repeated operator loops.

Login once, inspect context, perform targeted writes, and keep raw endpoint exploration separate from the stable command surface.

01
Authenticate locally

Use the browser flow; only the Dida365 `t` cookie is stored under `~/.dida-cli`.

02
Build context

Use `project list`, `folder list`, `tag list`, `+today`, `upcoming`, and `completed today` before writing.

03
Write deliberately

Preview generated changes with `--dry-run`; reserve `--yes` for destructive operations.

Commands that map to real resources.

No generic write tunnel by default. High-level verbs cover the jobs agents need most often. 

dida auth status --verify --json
dida project list --json
dida folder list --json
dida tag list --json
dida task upcoming --days 14 --json
dida completed list --from 2026-05-01 --to 2026-05-09 --json
dida task create --project <id> --title "Plan review" --json
dida project create --name "Agent staging" --dry-run --json
dida task delete <task-id> --project <id> --yes --json
