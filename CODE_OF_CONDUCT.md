# Code of Conduct

DidaCLI is a small open-source project for people building command-line and agent workflows around Dida365 / TickTick. Keep discussion technical, direct, and respectful.

## Expected Behavior

- Stay on the issue, pull request, or design question being discussed.
- Assume public comments can be read by new contributors later.
- Disagree with code, behavior, docs, or release choices without attacking the person.
- Do not post cookies, tokens, private task exports, screenshots with personal data, or full API response dumps.
- Use the private security advisory path for vulnerabilities instead of public issues.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Repeated off-topic comments after a maintainer asks to stop.
- Publishing another person's private data or credentials.
- Asking contributors to share secrets in chat, issues, pull requests, or logs.

## Enforcement

Maintainers may edit, hide, lock, or delete comments that violate this policy. They may also close issues, block accounts, or report abuse through GitHub when needed.

For security-sensitive reports, use the process in [SECURITY.md](SECURITY.md). For conduct problems that are not security issues, contact the maintainer privately or open a GitHub issue with only the public details needed to explain the problem.
